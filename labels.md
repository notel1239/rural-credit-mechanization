# Suggested Labels for Issues

The following labels should be created in the repository to organize questionnaire development and other project tasks.

## Questionnaire‑related
- `questionnaire` – General label for any issue about the survey instrument.
- `demographics` – For questions about household and farm characteristics (Section A).
- `finance` – For questions about credit access (Sections B and C).
- `dependent variable` – For issues concerning the machinery acquisition variable (Section D).

## Project Management
- `planning` – High‑level project planning, timelines, resources.
- `data collection` – Issues related to field work, enumerator training, sampling.
- `data cleaning` – Tasks for data processing, validation, and preparation.
- `analysis` – Econometric modeling, hypothesis testing, result interpretation.
- `documentation` – Writing reports, papers, README updates.

## Status
- `blocked` – Waiting on external input or resolution of another issue.
- `help wanted` – Requires input from other team members.
- `good first issue` – Suitable for new contributors.

## How to Create Labels
1. Go to the repository on GitHub.
2. Click **Issues** → **Labels** (or navigate to `https://github.com/notel1239/rural-credit-mechanization/labels`).
3. Click **New label** and fill in the name, description, and color.
4. Repeat for each label listed above.
